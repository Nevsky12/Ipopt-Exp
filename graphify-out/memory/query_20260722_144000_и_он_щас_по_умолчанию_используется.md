# Q: и он щас по умолчанию используется?

## Answer

Expanded from original query via graph vocab: [fgmres, refinement, pdfullspacesolver, candidate, structured, backend, full, kkt, residual, adaptive, default, canary]. Нет: в штатном PDFullSpaceSolver по умолчанию по-прежнему выбран iterative-refinement. FGMRES является opt-in: linear_system_refinement=fgmres-kdelta включает K_delta/M=K_delta, а fgmres-k включает K/M=K_delta. После выбора FGMRES действительно заменяет обычный refinement, стартует с direct x0 и повторно использует текущую факторизацию без новой факторизации. Отдельный adaptive FGMRES в C++23 structured candidate-first маршруте подключен автоматически, но запускается только при провале первичной проверки full-KKT residual; на естественных CAR2/DTOC3 было 0 срабатываний.
