# Q: Ок, теперь переанализируй алгоритм эффективной реструктурирования матрицы ККТ в случае если у задачи есть структура. Также проанализируй тезис /home/nevskys/Yandex.Disk/RocketOptimizer/09 Источники/Библиотека/_Файлы/Дополнение UchortMaterials/Kardos 2020 - High-Performance Interior Point Methods - Dissertation.pdf. Какие идеи из него окажутся полезными для нас?

## Answer

Expanded: [structured, topology, stage, bordered, block, tridiagonal, kkt, factorization, schur, condensation, ordering, permutation]
Вывод: текущий путь — точный частный случай chain и chain+small-primal-border. Общий алгоритм должен быть кэшируемым KKT decomposition plan: точное устранение complementarity-блоков; condition-aware selective slack condensation; симметричная перестановка в локальные блоки и separator; chain/tree/independent local factors; sparse или hierarchical Schur; композиция inertia только при точных факторах; reconstruction и residual исходной восьмиблочной KKT. При приближенных факторах structured solve следует использовать как right preconditioner full-KKT FGMRES. Из Kardoš наиболее полезны full non-reduced KKT refinement, selective slack elimination, sparse augmented Schur formation, разделение symbolic/numeric фаз и cost gate по размеру separator. Нельзя переносить как универсальные фиксированный tau=1e-8, dense Schur, GPU/MPI и OPF speedups. Приоритет: structured preconditioner+FGMRES; selective slack condensation; generalized separator metadata включая global duals; sparse/hierarchical Schur; затем parallel/GPU.
