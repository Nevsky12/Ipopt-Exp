# Q: Переанализировать Ipopt: современные численные методы, линейная алгебра и уменьшение ошибок floating-point

## Answer

Expanded from original query via vocab: [kkt, solver, factorization, refinement, residual, scaling, perturbation, inertia, structured, precision, numerical, complementarity]. Аудит показал, что ближайший приоритет — корректный normwise, затем componentwise full-KKT backward error, точный residual и автоматический direct-to-FGMRES-IR fallback. Постоянный FGMRES невыгоден на естественных CAR2 и DTOC3, но robust fallback полезен при плохой факторизации или регуляризации. Численное укрепление: LASSQ/hypot для составных норм, FMA и compensated reductions в residual и gate-скалярах, устойчивая рационализованная restoration-формула, безопасная BFGS scaling algebra, power-of-two symmetric equilibration. Структурный приоритет: stage/arrowhead/Riccati, batch RHS, pivoted small-block LDLT с 1x1/2x2 rook pivots без явной обратной матрицы, selective second elimination of inequality slacks с scale-relative threshold и full original KKT refinement. Kardoš подтверждает arrowhead Schur, sparse augmented local factors, selective slack elimination и обязательный original-KKT refinement. Mixed precision — после backward-error и condition telemetry. GPU — через device-resident HyKKT/LiftedKKT или крупный dense border, а не прямой cuDSS LDL с 1x1 pivots. Конкретные дефекты: ComputeResidualRatio игнорирует norm K; CompoundVector и structured Frobenius norms переполняются; restoration a+sqrt(a^2+b) катастрофически теряет точность; PardisoMKL sets invalid scaling value 2; solver API не сообщает rcond, pivot growth и perturbed pivots наружу.
